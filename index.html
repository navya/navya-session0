GIthub Pages




The idea behind github pages is free hosting for

static websites provided by github.com





What are static web sites?

Websites which have no database associated with them.

All the pages in such websites are simple HTML pages







Github provides one site per account/organisation and unlimited project sites




Project Website

Use it to showcase your project






Just create a  gh-pages branch  in your project repository  and push your website to this branch.





 

You can access the website by going to [username].github.io/[repo-name]

ex-  http://noraesae.github.io/perfect-scrollbar/

http://sofish.github.io/pen/






Account Website

Make you homepage/blog using this.








Create a repository with the name [username].github.io

Push your website to this repository.





You can access the website by going to the URL [username].github.io




Custom Domain

Create a file named CNAME in the root of your Page repository.







Put the domain (or subdomain) into the file.






Add  A records in your DNS pointing to 192.30.252.153, 192.30.252.154

Jekyll

Jekyll is a static site generator

http://jekyllrb.com/








Instead of writing everything in a WSYWIG linked to MySQL (like Wordpress or Blogger) you produce text files using Markdown which are then converted to static HTML









First, writing in Markdown is awesome.







Once you learn the syntax it’s incredibly fast and you don’t have to spend time playing with a tiny little editor window just to add some style to your posts






Second, writing in your favorite text editor is also awesome.






I write everything in Sublime Text 2 and every day I discover new tricks to make the process better. If you’ve ever had to write a blog post using one of those horrible little TinyMCE editors you will appreciate this feature






And lastly, static HTML is fast.
